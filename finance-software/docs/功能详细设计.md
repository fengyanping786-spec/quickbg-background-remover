## 2. 核心功能详细设计

### 2.1 账户管理

#### 2.1.1 功能流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  创建账户   │───▶│  账户列表   │───▶│  余额查询   │
└─────────────┘    └─────────────┘    └─────────────┘
       │                  │                  │
       ▼                  ▼                  ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  修改账户   │    │  流水记录   │    │  转账处理   │
└─────────────┘    └─────────────┘    └─────────────┘
```

#### 2.1.2 核心方法

```typescript
// account.service.ts
class AccountService {
  // 创建账户
  async create(data: CreateAccountDto): Promise<BankAccount>

  // 获取账户列表
  async findAll(tenantCode: string, page: number, pageSize: number): Promise<PaginatedResult>

  // 获取账户详情
  async findOne(id: string): Promise<BankAccount>

  // 更新账户
  async update(id: string, data: UpdateAccountDto): Promise<BankAccount>

  // 删除账户
  async remove(id: string): Promise<void>

  // 获取流水
  async getCashFlows(accountId: string, startDate?: string, endDate?: string): Promise<CashFlow[]>

  // 转账
  async transfer(fromId: string, toId: string, amount: number, remark?: string): Promise<void>
}
```

#### 2.1.3 业务规则

1. 账户类型：`bank`（银行）、`cash`（现金）、`wechat`（微信）、`alipay`（支付宝）
2. 转账时自动生成两条流水记录（转出+转入）
3. 删除账户时检查是否存在未结清流水

### 2.2 凭证管理

#### 2.2.1 凭证流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  凭证录入   │───▶│   凭证审核  │───▶│   凭证过账  │───▶│  财务报表   │
│  (draft)    │    │ (approved)  │    │  (posted)   │    │   生成     │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       │
       ▼
┌─────────────┐
│  使用模板   │
└─────────────┘
```

#### 2.2.2 核心方法

```typescript
class VoucherService {
  // 创建凭证
  async create(data: { voucher: Partial<Voucher>; items: Partial<VoucherItem>[] }, tenantCode: string): Promise<Voucher>

  // 使用模板创建凭证
  async createFromTemplate(templateId: string, amount: number, remark?: string): Promise<Voucher>

  // 审核凭证
  async approve(id: number, tenantCode: string): Promise<Voucher>

  // 过账凭证
  async post(id: number, tenantCode: string): Promise<Voucher>

  // 获取模板列表
  async getTemplates(tenantCode: string): Promise<VoucherTemplate[]>

  // 初始化默认模板
  async initDefaultTemplates(): Promise<void>
}
```

#### 2.2.3 业务规则

1. **借贷平衡**：借贷方金额必须相等，否则不允许保存
2. **至少一借一贷**：每张凭证至少有2条分录
3. **审核权限**：只有具有 `voucher:approve` 权限的用户可以审核
4. **过账规则**：已审核的凭证才能过账，过账后不能修改/删除

### 2.3 发票管理

#### 2.3.1 发票流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  OCR识别    │───▶│  发票录入   │───▶│  关联凭证   │───▶│  统计报表   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       │                  │                  │
       ▼                  ▼                  ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  真伪验证   │    │  发票查重   │    │  导出Excel  │
└─────────────┘    └─────────────┘    └─────────────┘
```

#### 2.3.2 核心方法

```typescript
class InvoiceService {
  // 创建发票
  async create(dto: CreateInvoiceDto): Promise<Invoice>

  // 分页查询
  async findAll(query: InvoiceQueryDto): Promise<{ data: Invoice[]; total: number }>

  // 发票查重
  async checkDuplicate(invoiceNo: string, amount: number): Promise<Invoice | null>

  // 关联凭证
  async bindVoucher(id: string, voucherId: string): Promise<Invoice>

  // 统计报表
  async getStatistics(startDate: string, endDate: string): Promise<any>
}

class OcrInvoiceService {
  // 识别增值税发票
  async recognizeVatInvoice(imageBuffer: Buffer): Promise<OcrInvoiceResult>

  // 验证发票真伪
  async verifyInvoiceTax(invoiceNo: string, invoiceCode: string, amount: number, date: string): Promise<any>
}
```

#### 2.3.3 业务规则

1. **查重规则**：发票号码 + 金额组合唯一
2. **税额计算**：税额 = 含税金额 × 税率 / (1 + 税率)
3. **OCR识别**：支持百度OCR，需要配置API Key
4. **真伪验证**：接入税务平台API（需要企业资质）

### 2.5 税务管理

#### 2.5.1 增值税计算

```
应纳税额 = 销项税额 - 进项税额 + 进项税额转出
税负率 = 应纳税额 / 销售金额 × 100%
```

#### 2.5.2 税负预警

| 税负率 | 状态 | 建议 |
|--------|------|------|
| < 1% | 偏低 | 可能存在税务风险 |
| 1% - 3% | 正常 | 合规范围 |
| > 3% | 偏高 | 请确认是否存在异常 |

#### 2.5.3 核心方法

```typescript
class TaxService {
  // 计算增值税

  // 保存税务记录

  // 税负预警检查
  async checkTaxBurdenWarning(taxRate: number): Promise<WarningResult>

  // 导出申报表
  async exportTaxDeclaration(id: string): Promise<DeclarationData>
}
```

### 2.6 权限管理

#### 2.6.1 权限模型

```
用户 ───── 角色 ───── 权限
(User)    (Role)    (Permission)
   │          │
   └──────────┘
   多对多关联
```

#### 2.6.2 权限验证流程

```
请求 ──▶ 认证中间件 ──▶ 权限守卫 ──▶ 控制器方法
         (JWT)         (RolesGuard)
                      │
                      ▼
            检查用户权限是否包含
            所需的 @RequirePermissions
```
