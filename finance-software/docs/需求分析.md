## 1. 项目概述

### 1.1 项目背景

随着小微企业数量的不断增长，这类企业往往没有专职的财务人员，日常财务核算主要依赖手工记账或简单的电子表格。这种方式存在以下问题：

1. **效率低下** - 手工记账费时费力，容易出错
2. **数据分散** - 财务数据分散在多个表格或纸质凭证中，难以汇总分析
3. **合规性差** - 缺乏规范的财务流程，难以满足税务要求
4. **决策支持不足** - 无法快速获取经营数据，影响决策效率

本系统旨在为10-50人规模的小微企业提供一套轻量级、易上手的财务管理系统，帮助企业规范财务流程、提升工作效率、降低出错风险。

### 1.2 项目目标

**核心目标**：简化日常财务操作，支持记账、报表、税务基础功能

**具体目标**：

1. 实现财务数据的电子化、规范化管理
2. 支持多账户、多账套管理，满足不同企业需求
3. 提供完整的凭证管理流程（录入、审核、过账）
4. 支持发票管理，包括OCR识别和真伪验证
5. 生成规范的财务报表（资产负债表、利润表）
6. 提供税务基础功能（增值税计算、税负预警）
7. 实现完善的用户权限管理
8. 支持数据备份和导出

### 1.3 目标用户

| 用户类型 | 特征 | 使用场景 |
|----------|------|----------|
| 小微企业 | 10-50人，无专职财务 | 日常记账、报表查看、税务申报 |
| 代理记账公司 | 为多家小微企业代理记账 | 多账套管理、集中处理 |
| 个体工商户 | 规模较小，业务简单 | 简单记账、发票管理 |

### 1.4 技术选型

| 层级 | 技术方案 | 理由 |
|------|----------|------|
| **前端** | Vue3 + Element Plus | 生态成熟，组件丰富，易上手 |
| **后端** | Node.js (NestJS) | 团队熟悉，异步性能好，部署方便 |
| **数据库** | PostgreSQL | 财务数据强一致性，支持多Schema |
| **ORM** | Prisma/TypeORM | 类型安全，迁移方便 |
| **部署** | Docker + Docker Compose | 便于本地/服务器统一环境 |

### 2.3 凭证管理模块

#### 2.3.1 功能描述

凭证管理是财务系统的核心模块，采用借贷记账法，支持凭证的录入、审核、过账全流程管理。

#### 2.3.2 功能列表

| 功能 | 描述 | 规则 |
|------|------|------|
| 凭证录入 | 借贷记账法，每笔至少一借一贷 | 金额必须相等 |
| 凭证模板 | 常用业务预设模板 | 差旅、工资、采购、销售 |
| 附件 | 凭证支持图片/PDF附件 | 最大10MB/个 |
| 审核 | 凭证需要复核才能入账 | 可配置是否需要审核 |
| 过账 | 凭证审核后更新科目余额 | 自动汇总到财务报表 |

#### 2.3.3 科目体系（简化版）

```

负债类 (2开头)
  - 2001 短期借款
  - 2202 应付账款
  - 2211 应付票据

所有者权益 (4开头)
  - 4001 实收资本
  - 4101 资本公积
  - 4103 利润分配
  - 4104 本年利润

损益类 (5/6开头)
  - 5001 主营业务收入
  - 6001 主营业务成本
```

|------|------|------|
