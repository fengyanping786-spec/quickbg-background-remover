## 2. 凭证管理详细设计

### 2.1 凭证数据结构

```typescript
// 凭证实体
interface Voucher {
  id: string;
  tenantId: string;
  year: number;
  voucherNo: string;           // 凭证编号
  voucherDate: Date;            // 凭证日期
  status: 'draft' | 'approved' | 'posted';  // 状态
  attachCount: number;         // 附件数量
  totalDebit: number;          // 借方合计
  totalCredit: number;         // 贷方合计
  attachment?: string;          // 附件路径
  templateType?: string;       // 模板类型
  makerId: string;
  makerName: string;
  checkerId?: string;
  checkerName?: string;
  posterId?: string;
  posterName?: string;
  remark?: string;
  periodId?: string;
  createdAt: Date;
  updatedAt: Date;
}

// 凭证分录
interface VoucherItem {
  id: string;
  voucherId: string;
  tenantId: string;
  year: number;
  lineNo: number;              // 行号
  accountCode: string;         // 科目编码
  accountName: string;         // 科目名称
  direction: 'debit' | 'credit';
  amount: number;              // 金额
  description?: string;        // 摘要
  auxInfo?: string;           // 辅助核算信息
}
```

### 2.2 凭证状态流转

```
草稿(draft) ──审核──> 已审核(approved) ──过账──> 已过账(posted)
     ▲              │                    │
     │              ↓                    ↓
     └────── 反审核 ←─────── 反过账 ←──────┘
```

### 2.3 凭证API接口

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/v1/vouchers | 凭证列表（分页、筛选） |
| POST | /api/v1/vouchers | 创建凭证（含分录） |
| GET | /api/v1/vouchers/:id | 凭证详情（含分录） |
| PUT | /api/v1/vouchers/:id | 更新凭证 |
| DELETE | /api/v1/vouchers/:id | 删除凭证 |
| POST | /api/v1/vouchers/:id/approve | 审核凭证 |
| POST | /api/v1/vouchers/:id/unapprove | 反审核 |
| POST | /api/v1/vouchers/:id/post | 过账 |
| POST | /api/v1/vouchers/:id/unpost | 反过账 |
| POST | /api/v1/vouchers/batch-approve | 批量审核 |
| POST | /api/v1/vouchers/batch-delete | 批量删除 |
| GET | /api/v1/vouchers/templates | 模板列表 |
| POST | /api/v1/vouchers/:id/attachments | 上传附件 |

### 2.4 凭证校验规则

1. **平衡校验**：借方合计 = 贷方合计
2. **科目存在**：科目编码必须在科目表中存在
3. **状态校验**：已审核的凭证不能修改；已过账的凭证不能删除
4. **断号检查**：凭证号不能重复

## 3. 科目管理详细设计

### 3.1 科目表结构

```typescript
interface Account {
  id: string;
  tenantId: string;
  year: number;
  code: string;             // 科目编码（4-10位）
  name: string;             // 科目名称
```

## 5. 期末处理详细设计

### 5.1 结转类型

| 结转类型 | 说明 | 凭证分录 |
|----------|------|----------|
| 结转收入 | 主营业务收入 → 本年利润 | 借：收入  贷：本年利润 |
| 结转成本 | 主营业务成本 → 本年利润 | 借：本年利润  贷：成本 |
| 结转费用 | 费用科目 → 本年利润 | 借：本年利润  贷：费用 |
| 计提折旧 | 固定资产折旧 | 借：管理费用  贷：累计折旧 |
| 结转增值税 | 增值税结转 | 借：销项税  贷：进项税  贷：应交增值税 |

### 5.2 结转流程

```
1. 检查本月凭证是否全部过账
2. 执行结转生成凭证
3. 凭证审核、过账
4. 月结标记
```

### 5.3 反结转

- 支持撤消未月结的结转凭证
- 反过账 → 反审核 → 删除

## 6. 报表详细设计

### 6.1 报表类型

| 报表 | 数据来源 | 公式 |
|------|----------|------|
| 资产负债表 | 科目余额 | 资产 = 负债 + 所有者权益 |
| 利润表 | 损益类科目发生额 | 收入 - 成本 - 费用 = 净利润 |
| 现金流量表 | 现金流量科目 | 经营 + 投资 + 筹资 |
| 科目余额表 | 所有科目 | 余额表 |
| 凭证汇总表 | 凭证分录 | 按凭证类型汇总 |

### 6.2 报表API

## 8. 数据隔离

### 8.1 账套隔离

所有业务表包含 `tenant_id` 字段，查询时自动过滤。

### 8.2 年度隔离

所有业务表包含 `year` 字段，支持同一账套不同年度数据独立。

### 8.3 期间隔离

会计期间表记录每个月的结账状态：
- `open` - 进行中
- `closed` - 已结账
- `year_closed` - 年末结账
