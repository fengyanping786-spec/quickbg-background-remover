# 2026-03-22 工作日志

## QuickBG 前端加载问题修复

### 问题描述
- 前端页面一直加载不出来
- F12 控制台显示网络错误

### 根本原因
1. **TypeScript 编译错误**：多处使用 `process.env.NODE_ENV`，在 Vite 生产环境需要用 `import.meta.env.MODE`
2. **API 路由配置错误**：`_routes.json` 把 `/api/*` 错误重定向到不存在的 `api.quickbg.ai`
3. **useMockApi 配置问题**：用户浏览器 localStorage 保存了旧的 `useMockApi: true`，导致使用 mock API 而非真实 API

### 修复内容
1. 修复 `analytics.ts`, `errorMonitor.ts`, `performanceMonitor.ts` 中的 `process.env` → `import.meta.env`
2. 修复 `_routes.json` API 路由指向正确的 worker 地址
3. 修复 `tsconfig.json` 严格模式，添加 `vite/client` 类型
4. 修复 `Uploader.tsx` 中不存在的 `Shrink` 图标 → `Minimize2`
5. 修复 `App.tsx` 中 `require()` 动态导入问题
6. **关键修复**：在代码中强制设置 `useMockApi = false`，防止本地存储的旧配置覆盖
